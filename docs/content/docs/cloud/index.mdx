## Introduction
PoloCloud is the simplest and easiest **Cloud for Minecraft Networks** based on Kotlin, designed to be fast, flexible, and simple.

PoloCloud has different parts:

<Cards>
    <Card icon={<CpuIcon className="text-purple-300" />} title='Agent'>
        Manages server instances and handles communication between the master node and individual Minecraft servers within the cloud infrastructure.
    </Card>

    <Card icon={<Network className="text-blue-300" />} title='Bridges'>
        Bridges handle the connection between proxy servers and game servers, enabling players to access game servers through the proxy network.
    </Card>

    <Card icon={<Layers className="text-green-300" />} title='Platforms'>
        Contains shared utilities, models, and common functionality used across all other components of the cloud system.
    </Card>

    <Card icon={<Braces className="text-red-300" />} title='Proto'>
        sd
    </Card>

    <Card icon={<Container className="text-cyan-300" />} title='Docker'>
        Handles containerization and deployment of PoloCloud services using Docker for isolated execution environments.
    </Card>
    
    <Card icon={<Workflow className="text-sky-300" />} title='Kubernetes'>
        Enables deployment and orchestration of PoloCloud services on Kubernetes clusters for scalable container management.
    </Card>
</Cards>
